Odoo庆祝自由。设计师可以自由地进一步自由地让用户根据自己的需求定制所有内容。

准备创建自己的主题？大。在开始之前，您应该了解以下一些事项。本教程是创建Odoo主题的指南。

网页设计师简介

如果您是第一次使用Odoo的网页设计师，那么您来对地方了。本简介将概述Odoo主题创建的基础知识。

注意

Odoo的团队创建了一个功能强大且易于使用的框架。没有必要知道使用这组工具的特殊语法。

从常见的CMS到Odoo

注意

如果你总是以同样的方式思考和工作，你可能会得到相同的结果。如果你想要一些全新的东西，那么尝试不同的东西。

我的header.php文件在哪里？

这通常是网页设计师用来使用Wordpress或Joomla并第一次来到Odoo的第一个问题。

实际上，在使用常见的CMS时，您必须编写几个文件（如header.php，page.php，post.php等），以便为您的网站创建基本结构。使用这些系统，此基础结构充当设计基础，您必须随时更新以确保CMS内的兼容性。因此，即使你花了几个小时编写文件，你还没有开始设计。

这并不适用于创建Odoo主题。

注意

我们认为主题设计应该简单（而且功能强大）。当我们创建网站构建器时，我们决定从头开始，而不是依赖已经存在的东西。这种方法让我们可以自由地专注于对设计师非常重要的事情：风格，内容和背后的逻辑。没有更多的技术挣扎。

Odoo默认主题结构

Odoo带有默认主题结构。这是一个非常基本的“主题”，提供最小的结构和布局。当您创建新主题时，实际上是在扩展它。实际上，它总是在您的设置中启用，它的行为与我们上面提到的CMS的基本结构完全相同，只是您不必创建或维护它。它将在您的Odoo安装中自动升级，并且由于它包含在Website Builder模块中，因此默认情况下一切都是顺利集成的。

因此，您可以完全自由地专注于设计，而此结构可以提供集成和功能。

主要特点：

  • 页面，博客和电子商务的基本布局
  • 网站构建器集成
  • 基本片段
  • 自动减少/ Sass编译
  • 自动Js和CSS缩小和组合

主要技术：

  • Twitter Bootstrap
  • jQuery的
  • jQuery UI
  • underscore.js

思考“模块化”

Odoo主题不是包含HTML或PHP文件的文件夹，它是用XML编写的模块化框架。以前从未使用过XML文件？不用担心，在学完本教程后，您将能够仅使用HTML的基本知识创建第一个主题。

使用经典的Web设计工作流程，您通常会编写整个页面的布局。结果是“静态”网页。当然，您可以更新内容，但您的客户需要您进行基本更改。

为Odoo创建主题是对观点的彻底改变。您可以创建块（片段），而不是定义页面的完整布局，让用户选择“拖放”它们的位置，自己创建页面布局。我们称之为模块化设计。

想象一下Odoo主题作为您必须创建和样式的元素和选项的“列表”。作为设计师，您的目标是设计这些元素的样式，以便获得精彩的结果，无论最终用户选择放置它们的位置。

让我们来看看我们的“列表”元素：

片段（或构建块）

一段HTML代码。用户将使用我们内置的Website Builder界面进行拖放，修改和组合。您可以为每个片段定义选项和样式集。用户将根据他们的需要从中进行选择。

网页

这些是普通网页，除了它们可由最终用户编辑，并且您可以通过将片段拖入其中来定义用户可以“填充”的空白区域。

样式

使用标准CSS文件（或Less / Sass）定义样式。您可以将样式定义为默认样式或可选样式。默认样式在主题中始终处于活动状态，用户可以启用或禁用可选样式。

功能

由于Odoo的模块化，一切都可以更加个性化。这意味着您的创造力有无限可能。添加功能很简单，并且为最终用户提供可自定义的选项很简单。

Odoo的XML文件，概述

任何Odoo XML文件都以编码规范开头。之后，您必须在代码中编写代码<odoo>。

[XML]
 <？xml version =“1.0”encoding =“utf-8”？> 
<odoo>

    ##你的代码在这里

</ odoo>

几乎您创建的每个元素和选项都必须放在<template>标记内，就像本例中一样。

[XML]
 <template  id = “my_title”  name = “My title” > 
  <h1>这是一个HTML块</ h1> 
  <h2  class = “lead” >这是一个副标题</ h2> 
</ template>

重要

不要误解是什么template意思。模板标签仅定义一段html代码或选项 - 但它不一定与元素的视觉排列一致。

前面的代码定义了一个标题，但它不会在任何地方显示，因为该模板与Odoo默认结构的任何部分都没有关联。为此，您可以使用 xpath，qWeb或两者的组合。

继续阅读教程，了解如何使用您自己的代码正确扩展它。

由于XML文件仅在安装主题时加载，因此每次对xml文件进行更改时都必须强制重新加载。

为此，请单击模块页面中的“升级”按钮。

创建主题模块

Odoo的主题像模块一样打包。即使您正在为您的公司或客户设计一个非常简单的网站，您也需要像Odoo模块一样打包主题。

main folder
创建一个文件夹并将其命名为：theme_后跟主题名称。
__manifest__.py
创建一个空文档并将其保存到您的文件夹中 __manifest__.py。这将包含主题的配置信息。
__init__.py
创建另一个空文件并命名__init__.py。这是一个强制性的系统文件。创建并将其留空。
views和static文件夹
在主文件夹中创建它们。在views您需要将定义你的片断，您的网页和你的选择你的XML文件。static文件夹是您的样式，图像和自定义js代码的正确位置。

重要

在odoo和init文件名的开头和两个末尾使用两个下划线字符。

Odoo页面的结构

Odoo页面是两种元素，跨页面和唯一的组合的视觉结果。默认情况下，Odoo为您提供页眉和页脚（跨页）以及包含使您的页面唯一的内容的唯一主要元素。

注意

跨页元素在每个页面上都是相同的。唯一元素仅与特定页面相关。

要检查默认布局，只需使用网站构建器创建新页面。单击内容‣新页面并添加页面名称。使用浏览器检查页面。

< div  id = “wrapwrap” > 
  < header  /> 
  < main  /> 
  < footer  /> 
</ div >

 </ odoo>

添加样式

当然，如果您想提供独特的设计，Bootstrap是不够的。以下步骤将指导您如何向主题添加自定义样式。最终的结果不会很好，但会为您提供足够的信息来自己构建。

        text-align ： center ; 
        margin-bottom ： 1em ; 
        字体大小： 1 .5em ;
    } 
}

我们的文件已准备就绪，但尚未包含在我们的主题中。

<template  id = “mystyle”  name = “My style”  inherit_id = “website.assets_frontend” > 
    <xpath  expr = “link [last（）]”  position = “after” > 
        <link  rel = “stylesheet”  type = “text / scss“  href = ”/ theme folder / static / scss / style.scss“ /> 
    </ xpath> 
</ template>

我们刚刚创建了一个指定scss文件的模板。如您所见，我们的模板有一个特殊的属性叫做inherit_id。该属性告诉Odoo我们的模板是指另一个模板以便操作。

在这种情况下，我们指的是assets_frontend位于website模块中的模板。assets_frontend指定网站构建器加载的资产列表，我们的目标是将我们的scss文件添加到此列表中。

这可以通过使用与属性的XPath实现 expr="link[last()]"和position="after"，这意味着“ 把我的风格文件，并将其放置在资产列表中的最后一个环节后 ”。

将它放在最后一个之后，我们确保我们的文件将在最后加载并优先考虑。

最后在__manifest__.py文件中添加assets.xml。

我们的scss文件现在包含在我们的主题中，它将自动编译，缩小并与所有Odoo的资产相结合。

代码段选项

选项允许发布者使用网站构建器的UI编辑代码段的外观。使用网站构建器功能，您可以轻松创建代码段选项并自动将其添加到UI。

选项组属性

选项包含在组中。组可以具有定义所包含选项如何与用户界面交互的属性。

data-selector="[css selector(s)]"
将包含在组中的所有选项绑定到特定元素。
data-js=" custom method name "
用于绑定自定义Javascript方法。
data-drop-in="[css selector(s)]"
定义可以放入代码段的元素列表。
data-drop-near="[css selector(s)]"
定义可以在其旁边删除代码段的元素列表。

默认选项方法

选项将标准CSS类应用于代码段。根据您选择的方法，UI的行为会有所不同。

data-select-class="[class name]"
同一组中的更多data-select-class定义了用户可以选择应用的类列表。一次只能启用一个选项。
data-toggle-class="[class name]"
data-toggle-class用于将列表中的一个或多个CSS类应用于代码段。可以一次应用多个选择。

SEO最佳实践

促进内容插入

现代搜索引擎算法越来越关注内容，这意味着对关键字饱和度的关注较少，并且更关注内容是否与关键字实际相关。

由于内容对SEO非常重要，因此您应该专注于为发布者提供轻松插入的工具。重要的是，您的代码段“内容响应”，这意味着它们应该适合发布者的内容，无论大小。

让我们来看看这个经典的两列代码片段示例，它以两种不同的方式实现。

坏

使用固定图像，发布者将被迫限制文本以遵循布局。

好

使用符合列高的背景图像，无论图像的高度如何，出版商都可以自由添加内容。

页面分割

基本上，页面分段意味着页面被分成几个单独的部分，并且这些部分被搜索引擎视为单独的条目。设计页面或片段时，应确保使用正确的标签以便于搜索引擎索引。

<article>
指定独立的内容块。在它内部应该是一个独立的内容，它本身应该是有意义的。您可以将<article>元素嵌套在另一个中。在这种情况下，暗示嵌套元素与外部<article>元素相关。
<header>
表示自包含内容块的标题部分（an <article>）。
<section>
是片段默认标记，它指定内容块的子部分。它可用于将<article>内容分成几个部分。建议使用标题元素（<h1>- <h6>）来定义节的主题。
<hgroup>

用于包裹标题的一部分（<h1>- <h6>）。一个很好的例子是顶部标题和副标题的文章：

< hgroup > 
  < h1 >主标题</ h1 > 
  < h2 >小标题</ h2 > 
</ hgroup >

描述你的页面

定义关键字

您应该为这些关键字使用适当的相关关键字和同义词。您可以使用顶部栏中的内置“提升”功能为每个页面定义它们。

定义标题和描述

使用“提升”功能定义它们。保持页面标题简短，并包含页面的主要关键字短语。好的标题唤起了情绪反应，提出问题或承诺某些事情。

描述虽然对搜索引擎排名不重要，但对于获得用户点击非常重要。这是一个广告内容的机会，让人们可以准确地搜索给定页面是否包含他们正在寻找的信息。重要的是每页上的标题和描述都是唯一的。
